## 2. 简历模板说明

### 整体布局

采用**左右两栏布局**，左侧约 270px 为信息栏，右侧为主内容区：

```
┌─────────────────────────────────────────────┐
│  ┌──────────────┬──────────────────────────┐  │
│  │   [头像]      │  姓名 + 职位 + GitHub     │  │
│  │              ├──────────────────────────┤  │
│  │  姓名/职位    │  个人简介                  │  │
│  ├──────────────┤  项目经历                  │  │
│  │  联系方式      │    · 项目1               │  │
│  │  专业技能      │    · 项目2               │  │
│  │  证书荣誉      │    · 项目3               │  │
│  │              │  教育背景                  │  │
│  │              │  在校经历                  │  │
│  └──────────────┴──────────────────────────┘  │
└─────────────────────────────────────────────┘
```

### 配色方案

| 用途 | 颜色值 | 说明 |
|------|--------|------|
| 页面背景 | `#eef0f2` | 浅灰背景 |
| 左侧栏背景 | `#f8f9fb` | 比主背景略浅 |
| 卡片背景 | `#ffffff` | 白色 |
| 主色（蓝） | `#2563eb` | 标题、标签、装饰点 |
| 正文 | `#1e293b` | 深色文字 |
| 辅助文字 | `#64748b` | 灰色说明文字 |

### 字体

- 主字体：**Sora**（Google Fonts 加载）
- 后备字体：`Segoe UI`, sans-serif

### 核心技术

- 纯 HTML + CSS（无 JavaScript 框架依赖）
- CSS Grid 实现双栏布局
- CSS 变量统一管理配色
- `@media print` 实现打印优化
- `window.print()` 浏览器原生导出PDF

### 3.5 准备头像照片

```powershell
# 将照片复制到项目目录并重命名为 photo.jpg
Copy-Item "E:\code\resume\证件照.jpg" "E:\code\resume\photo.jpg" -Force
```

### 3.6 初始化 Git 仓库

```powershell
# 初始化 git 仓库
cd E:\code\resume
git init

# 创建 main 分支
git checkout -b main

# 添加所有文件并提交
git add -A
git commit -m "feat: 初始化个人简历页面"
```

### 3.7 创建 GitHub 仓库并推送

```powershell
# 使用 gh 创建远程仓库（用户名.dghc2023.io 形式才能直接访问）
gh repo create GitHub用户名/GitHub用户名.github.io --public --description "个人简历"

# 添加远程仓库并推送
git remote add origin https://github.com/GitHub用户名/GitHub用户名.github.io.git
git push -u origin main
```

> **注意**：如果仓库名设置为 `用户名.github.io`，则 GitHub Pages 会自动启用，
> 访问链接为 `https://用户名.github.io/`。
>
> 如果使用其他仓库名（如 `resume`），访问链接为 `https://用户名.github.io/仓库名/`，
> 且需要在仓库 Settings > Pages 中手动设置 Source 为 main 分支。

### 3.8 更新简历内容

修改完 `index.html` 后，提交并推送即可自动部署：

```powershell
cd E:\code\resume
git add index.html
git commit -m "feat: 更新简历内容"
git push
```

等待约 1-2 分钟，访问 `https://GitHub用户名.github.io/` 即可看到更新。

在浏览器打印对话框中选择 **「另存为 PDF」** 即可。

## 5. 常见问题

### Q: 页面更新后没有变化？

GitHub Pages 有 CDN 缓存，推送后约 1-2 分钟生效。
可以在 URL 后加 `?v=版本号` 强制刷新：`https://xxx.github.io/?v=2`

### Q: 如何查看部署状态？

```powershell
gh api repos/GitHub用户名/GitHub用户名.github.io/pages --jq '.status'
```

返回 `built` 表示部署完成。

### Q: 头像不显示？

检查 `photo.jpg` 文件是否在项目根目录，且文件名与 `index.html` 中 `src` 一致。
注意文件名大小写（Windows 不区分但 GitHub Pages 的 Linux 服务器区分）。

### Q: 手机端显示混乱？

模板已内置响应式代码，屏幕宽度小于 680px 时会自动变为单栏布局：

```css
@media(max-width:680px){
  .resume{grid-template-columns:1fr}  /* 双栏变单栏 */
  .sidebar{border-right:none; ...}     /* 去掉分割线 */
}
```
